# ADR-0001：Electron 桌面架构

## 状态

已接受。

## 背景

“纸间”原本是一个依赖浏览器 `localStorage` 的单文件网页。桌面版需要可靠的离线持久化、原生文件操作、托盘、全局快捷键和自动更新，同时保持私人笔记不离开本机。

## 决策

- 使用 Electron、React 与 TypeScript，主进程、预加载脚本和渲染进程严格分层。
- 使用 SQLite 作为唯一持久化数据源，主进程独占数据库访问。
- 渲染进程关闭 Node 集成并启用上下文隔离与沙箱，只通过经过校验的类型化 API 调用桌面能力。
- 私有源码仓库使用 `Kiko3127/zhijian-desktop`；公开安装包和更新元数据发布到 `Kiko3127/zhijian-releases`，客户端不携带 GitHub 凭据。

## 后果

获得事务、迁移、崩溃恢复和可测试的安全边界；代价是需要维护 IPC 合同、SQLite 原生模块重建以及 Windows 打包发布流程。

## 备选方案

- 继续使用 `localStorage`：迁移简单，但桌面数据管理和恢复能力不足。
- JSON 文件：直观但并发写入、查询和迁移较弱。
- 私有 GitHub Releases 自动更新：需要在客户端嵌入凭据，因安全风险拒绝。
